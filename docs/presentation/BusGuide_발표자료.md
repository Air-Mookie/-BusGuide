# BusGuide — 어르신 음성 기반 버스 안내 시스템

---

## 1. 문제 정의 (2분)

### 디지털 소외 — 숫자로 보기
- 부산시 70대 이상 인구: **35만 명**
- 스마트폰 미사용자: **24만 명 (68%)**
- 버스 앱 사용 가능한 70대: **12%**

### 현장의 불편함
- 어르신들이 매번 기사님께 "어디 가요?" 물어봄
- 운전 중 기사 방해 → **안전 문제**
- 기존 BIS 전광판: 노선번호만 표시, 목적지 안내 없음

### 우리가 해결하려는 것
> **스마트폰을 못 쓰는 어르신도 음성으로 버스 정보를 얻을 수 있게**

## 2. 솔루션 소개 (3분)

### 한 문장 요약
> 정류장 태블릿에 **"명지병원 가는 버스 있어요?"** 라고 말하면
> **"58번 버스가 5분 후에 옵니다"** 라고 알려주는 시스템

### 핵심 기능 5가지
1. **음성 질의 → 음성+화면 답변** (말만 하면 끝)
2. **연속 대화** ("거기서 해운대 가려면?" — 이전 맥락 기억)
3. **부산 사투리 인식** ("어데 카는 버스예요?" 처리)
4. **터치 백업** (음성 실패 시 큰 버튼으로)
5. **시니어 특화 UX** (48pt 폰트, 고대비, 삼중 피드백)

### 데모 시나리오
```
어르신: [말하기 버튼 터치]
시스템: "어디 가시려고요?"

어르신: "명지병원 카는 버스 있나예?"
시스템: (사투리 자동 변환)
        "명지병원 가시려면
         58번 버스 타시면 됩니다.
         5분 후 도착 예정입니다."

어르신: "거기서 해운대 가려면?"
시스템: (컨텍스트: 명지병원 기억)
        "181번 버스로 갈아타시면
         해운대에 갈 수 있습니다."
```

## 3. 기술 아키텍처 (3분)

### 심플 클라이언트 패턴

```
태블릿 (Android)          FastAPI 서버           TAGO API
┌──────────────┐     ┌──────────────────┐   ┌──────────────┐
│ STT 음성인식  │────→│ ① 사투리 변환     │──→│ 실시간       │
│ TTS 음성출력  │←────│ ② 의도 파악       │←──│ 도착정보     │
│ 시니어 UI    │     │ ③ 유사어 매칭     │   │ 노선정보     │
│ 키오스크 모드 │     │ ④ TAGO API 호출   │   │ 정류장 정보  │
│              │     │ ⑤ 컨텍스트 저장   │   │              │
│              │     │ ⑥ 응답 생성       │   │              │
└──────────────┘     └──────────────────┘   └──────────────┘
```

### 기술 스택

| 파트 | 기술 |
|------|------|
| **Android 앱** | Kotlin, Jetpack Compose, Hilt, Retrofit |
| **백엔드 서버** | FastAPI (Python), Kiwi (한국어 NLP) |
| **공공 데이터** | 국토교통부 TAGO 전국 버스정보 API |
| **음성** | Android SpeechRecognizer (STT), TextToSpeech (TTS) |

### 클린 아키텍처

```
Android 앱 구조:
├── presentation/  ← UI (Compose 화면)
├── domain/        ← 비즈니스 로직 (모델, 인터페이스)
└── data/          ← 데이터 접근 (Retrofit API 호출)

서버 구조:
├── routers/       ← API 엔드포인트
├── services/      ← NLP, 버스, 컨텍스트 서비스
└── data/          ← JSON 설정 파일 (하드코딩 방지)
```

## 4. 핵심 기술 시연 (5분)

### 4-1. 사투리 변환 엔진

**직접 구현한 NLP 파이프라인:**

```
입력: "명지병원 카는 버스 있나예?"
      ↓ dialect_map.json 기반 변환
출력: "명지병원 가는 버스 있나요?"
```

- 19개 부산 사투리 매핑 (JSON 파일 — 코드 수정 없이 확장)
- 긴 키워드 우선 치환 ("있나예" > "나예" 충돌 방지)

### 4-2. 의도 분류

| 의도 | 키워드 예시 | 우선순위 |
|------|-----------|---------|
| 환승_안내 | "갈아타", "갔다가" | 1 (최우선) |
| 도착_시간 | "몇 분", "언제" | 2 |
| 정류장_확인 | "여기 어디", "무슨 버스" | 3 |
| 노선_검색 | "가는 버스", "가려면" | 4 (기본값) |

### 4-3. 유사어 매칭

```
"바다"       → "해운대"
"기차역"     → "부산역"
"자갈치"     → "남포동"
"명지 큰병원" → "명지병원"
```

- 10개 주요 지명 + 40+ 유사어 (synonyms.json)
- 긴 키워드 우선 매칭 (부분 매칭 충돌 방지)

### 4-4. 컨텍스트 대화

```
질문 1: "58번 어디 가요?"
→ 컨텍스트 저장: {destination: "명지병원", bus: "58"}

질문 2: "몇 분 후에 와요?"
→ 58번 기억됨! "5분 후 도착합니다"

질문 3: "거기서 해운대 가려면?"
→ "거기" = 명지병원 (대명사 해석)
→ "181번 버스로 갈아타세요"
```

- 세션 ID 기반 인메모리 저장 (5분 TTL 자동 만료)

### 4-5. 시니어 UX

| 항목 | 일반 앱 | BusGuide |
|------|---------|----------|
| 폰트 | 14sp | **48sp** (3.4배) |
| 버튼 | 48dp | **120dp** (2.5배) |
| 색상 | 다양 | **고대비** (검정+노랑) |
| 피드백 | 시각만 | **음성+시각+진동** |

## Q&A

### 예상 질문

**Q: "이미 버스 앱에 다 있는데?"**
> A: 타겟이 다릅니다. 70대 이상 버스 앱 사용률 12%. 저희는 **앱을 못 쓰는 24만 명**을 위한 것입니다.

**Q: "소음 때문에 음성인식 안 되지 않나?"**
> A: 음성 + 터치 병행 (멀티모달). 3초 인식 실패 시 자동으로 터치 화면 전환.

**Q: "창의성이 부족한 것 같은데?"**
> A: 세 가지 창의성:
> 1. 직접 구현한 사투리 NLP 엔진
> 2. 컨텍스트 인식 연속 대화
> 3. 시니어 특화 멀티모달 UX

**Q: "1년 안에 완성 가능?"**
> A: 서버 이미 29개 테스트 통과, 앱 구조 완성. 기존 API(TAGO, SpeechRecognizer) 활용으로 밑바닥부터 만들 필요 없음.
